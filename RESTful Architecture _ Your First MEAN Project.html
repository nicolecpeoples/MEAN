RESTful Architecture

Consistent naming keeps big projects organized.

Large projects can easily become a morass of confusing function names, especially in projects in stacks like the MEAN stack where data are passed as objects throughout the project. Using standardized design principles throughout a project can help prevent confusing code. A common starting point for standardization for projects is RESTful routing as described here. This is just a starting point for many projects. For enterprise level projects, a good style guide will extend all the way through how variables should be named and placed in the code, and more!

Verb Example Path Factory(client side) Route Controller(server side) what it does and notes:
get '/friends' factory.index() app.get('/friends', friends.index); index:function(req,res)... returns all friends from db, and often loads a view showing these data
get '/friends/123123' factory.show() app.get('/friends/:id', friends.show) show:function(req,res)... returns one friend that has an _id that matches req(uest).params.id
post '/friends' factory.create() app.post('/friends', friends.create) create:function(req,res)... creates a new friend in the db, from info in req(uest).body
put/patch '/friends/12313' factory.update() app.put/patch('/friends/:id', friends.update) update:function(req,res)... updates the friend that has an _id from req(uest).params.id with data passed in from req(uest).body
delete '/friends/12313' factory.delete() app.delete('/friends/:id', friends.delete) delete:function(req,res)... deletes a friend that has an _id from req(uest).params.id
get '/friends/new' handled by partials in Angular would be: app.get('/friends/new', function (req,res){res.render('friends/new.html')}); would be:
new:function(req,res)... 		Creates a form that can be filled out to make a new friend, and passed to the create method
get  '/friends/12313/edit' handled by partials in Angular would be:
app.get('/friends/:id/edit', function (req,res){res.render('friends/edit.html')}); 		would be:
edit:function(req,res)... 		Creates a form that can be filled out to update a specific friend with an id.  (you might pre-populate the form with a show, using the :id)...
